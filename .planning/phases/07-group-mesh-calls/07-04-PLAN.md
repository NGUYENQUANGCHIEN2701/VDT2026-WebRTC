## Phase Goal

**As a** user entering a group call, **I want** a clear invite flow and a stable grid-based room UI,
**so that** the group feature feels real without compromising the existing 1-1 call experience.

<objective>
Implement the full Phase 7 frontend experience using the exact UI contract from `07-UI-SPEC.md`:
multi-invite entry from the online list, incoming/outgoing invite surfaces, a separate
`/group-call` route, participant tiles, room leave controls, and per-peer debug visibility for
the bitrate cap. This plan is where the additive room flow becomes user-facing.

The implementation must stay faithful to D-01, D-05, and D-06, and it must not degrade the
existing `/call` route or its debug panel mode.

Scope note: this plan intentionally touches 12 UI files because the approved `07-UI-SPEC.md`
binds invite selection, toast feedback, route wiring, room tiles, controls, and debug visibility
into one user-facing slice. The work is split into two tasks so the invite/toast path and the
`/group-call` page/debug path remain independently verifiable.
</objective>
