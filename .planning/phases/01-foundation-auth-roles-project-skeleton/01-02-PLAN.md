## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /api/auth/login | Untrusted credentials; SQL injection risk via JPA params |
| Browser → POST /api/auth/register | Untrusted user input; email/username uniqueness constraint bypass attempt |
| Authorization header → JwtAuthFilter | Token may be forged or expired |
| Set-Cookie header → Browser | Cookie flags must be correct for XSS/CSRF protection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | POST /api/auth/login | mitigate | `BadCredentialsException` for both unknown user AND wrong password — same 401 response (no enumeration, V7). Generic message "incorrect username or password" enforced in `GlobalExceptionHandler`. |
| T-02-02 | Spoofing | JWT token in Authorization header | mitigate | `JwtAuthFilter` calls `jwtService.extractAllClaims()` which throws `JwtException` on invalid signature or expiry; filter returns 401 and aborts chain without setting auth. |
| T-02-03 | Tampering | SQL injection via register/login DTOs | mitigate | Spring Data JPA uses parameterized queries exclusively; never string-concatenated SQL. `@Valid` on DTOs validates before any DB call. |
| T-02-04 | Information Disclosure | Access token storage on client | mitigate | Access token stored in JS module-level variable only (D-03) — NOT localStorage, NOT sessionStorage. Token never touches the DOM. 15-min TTL limits exposure window. |
| T-02-05 | Information Disclosure | httpOnly cookie (refresh token) | mitigate | `ResponseCookie` with `httpOnly=true`, `SameSite=Lax`, `path=/api/auth`, `secure=false` (Phase 1 HTTP; set to true in Phase 3 HTTPS). JS cannot read the refresh token (D-01). |
| T-02-06 | Tampering | CSRF on /api/auth/refresh | mitigate | `SameSite=Lax` cookie prevents cross-site POST from third-party origin. Stateless JWT means no CSRF token needed for other endpoints. |
| T-02-07 | Elevation of Privilege | Weak JWT secret | mitigate | JWT_SECRET loaded from env var only (never hardcoded). Minimum 32-char base64 key enforced by `Keys.hmacShaKeyFor()` which throws if key is too short for HS256. |
| T-02-08 | Tampering | Concurrent 401 rotation collision | mitigate | `isRefreshing` + `failedQueue` pattern in axiosInstance.ts prevents multiple simultaneous refresh calls (Pitfall 4). Without this, token rotation can loop and lock out the user. |
| T-02-SC | Tampering | npm package supply chain | mitigate | All packages pre-audited in RESEARCH.md Package Legitimacy Audit (all Approved via `npm view`). No `[ASSUMED]` or `[SUS]` packages in this plan. |
