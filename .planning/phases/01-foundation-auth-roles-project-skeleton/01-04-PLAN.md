## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JWT Bearer token → AdminController | ADMIN role in JWT claims must be verified server-side |
| React route guard → /admin | Client-side guard is UI convenience only; server enforces the real boundary |
| .env file → docker-compose.yml | JWT_SECRET must not be committed to version control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | GET /api/admin/dashboard | mitigate | Dual enforcement: (1) `SecurityFilterChain` `requestMatchers("/api/admin/**").hasRole("ADMIN")` — blocks at filter level; (2) `@PreAuthorize("hasRole('ADMIN')")` on `AdminController` class — method-security second layer. Both layers require `@EnableMethodSecurity` + valid ADMIN JWT. SecurityConfigTest verifies the 403 for USER role. |
| T-04-02 | Elevation of Privilege | React /admin route with USER role | mitigate | `ProtectedRoute requiredRole="ADMIN"` redirects USER role to `/` silently. Client-side guard prevents UI access. Server-side enforcement (`T-04-01`) is the real boundary. |
| T-04-03 | Information Disclosure | JWT_SECRET in docker-compose.yml | mitigate | Default placeholder `change-this-in-production-min-32-chars-base64` warns operator. `docs/setup.md` instructs to generate fresh secret via `openssl rand -base64 32` and store in `.env`. `.env` added to `.gitignore` (executor must confirm this). |
| T-04-04 | Information Disclosure | Seed admin default password in setup.md | accept | Seed password (Admin@123) is documented in setup.md for developer convenience. It is a dev seed, not production. Production deployments MUST change this. Documenting it is required for the INFR-07 deliverable. |
| T-04-05 | Information Disclosure | Swagger UI exposing API surface | mitigate | Swagger UI at `/swagger-ui.html` is useful during development. For production hardening (Phase 9), add `springdoc.api-docs.enabled: false` to the production profile. Phase 1 accepts it open for dev convenience. |
</threat_model>
