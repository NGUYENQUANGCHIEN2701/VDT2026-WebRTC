<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client reconnect → server resync | Server re-validates call ownership before resync-ack; client trusts authoritative state |
| sessionStorage callId | Local rejoin hint; server CAS validates the callId still exists and the principal is a participant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Spoofing | sessionStorage callId rejoin (inject someone else's callId) | mitigate | state-resync handler (Plan 03) reads call:{callId} and only resyncs if the WS principal is callerId or calleeId; a foreign callId yields no active call → client gets ended/empty, no media rejoin |
| T-04-06-02 | DoS | grace-period abuse via reconnect loop | mitigate | Grace timer is server-owned and bounded by Redis TTL 300s (Plan 02/03); client reconnect cannot extend beyond the hard ceiling |
| T-04-06-03 | Tampering | client falsely staying 'connected' after real drop | mitigate | The server grace timer is authoritative: if the client never resyncs, the server transitions to dropped and broadcasts ended{dropped}; the client cannot override the server's terminal state |
| T-04-06-04 | Information Disclosure | resync-ack presence snapshot | accept | Presence snapshot is already broadcast to all online users (Phase 2); resync-ack reuses the same data, no new disclosure |
| T-04-06-SC | Tampering | npm/pip/cargo installs | accept | No new npm packages (zero new frontend deps per 04-RESEARCH.md audit); native restartIce + sessionStorage only |
</threat_model>
