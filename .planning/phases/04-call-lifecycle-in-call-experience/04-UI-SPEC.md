## Color

60/30/10 role mapping for the call UI context (full-screen, dark video backdrop):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000` / `var(--bg)` | Video area background (black for media canvas), call-page outer container (`var(--bg)`) |
| Secondary (30%) | `var(--code-bg)` | IncomingCallCard card, SelfViewPreview card, call-summary card, reconnect overlay card area |
| Accent (10%) | `var(--accent)` | Reserved — see below |
| Destructive | `#dc2626` | Reject button, hang-up button, dropped end-reason warning text, failed connection indicator |

**Accent (`var(--accent)`) reserved for:**
- None — accent color is NOT used in Phase 4 call UI components. Call UI uses semantic greens/reds for lifecycle actions. Accent (`#aa3bff`) is the app-level brand color used only on non-call pages (counters, links). This is intentional — keep the in-call palette to green/red/neutral to match the gravity of communication UI.

**Warning semantic color (`#d97706` amber):**
- RTT > 300ms threshold indicator (carry-forward from Phase 3 QualityIndicator)
- `dropped` end-reason is NOT amber — use `#dc2626` red (it is an error, not a warning; D-08 says "màu cảnh báo" but in the existing palette, red is the correct signal for dropped/failed)

## Interaction Contracts

### Incoming Call Screen (CALL-02)

- Trigger: `callState === 'incoming'` (server broadcasts `call-state-changed` → client FSM transitions to `incoming`)
- Renders: `IncomingCallCard` modal overlay (existing) + starts `<audio autoPlay loop>` ringtone
- Ring audio: `/public/ringtone.mp3` — executor must add a short (1-2s looping) ringtone asset to `frontend/public/`
- Accept: fires intent `call-accept` to server; stops audio; navigates to `/call`
- Reject: fires intent `call-reject` to server; stops audio; stays on current page (Home)
- Ring timeout (~30s): server fires `call-state-changed { reason: 'missed' }` → client stops audio, shows "Bạn đã nhỡ cuộc gọi từ X" toast, remains on Home

### Outgoing Call / Busy (CALL-03, CALL-05)

- Trigger: user clicks "Gọi" on Home → client sends `call-invite` intent
- If callee busy: server responds immediately without ringing callee; client receives `call-state-changed { reason: 'busy' }` and shows toast "X đang bận" on Home — does NOT navigate to `/call`
- If callee available: client transitions to `outgoing`, navigates to `/call`, shows `SelfViewPreview`
- Cancel: fires intent `call-cancel` to server; transitions to `ended { reason: 'cancelled' }` → `CallSummaryScreen`

### In-Call Control Bar (MEDIA-01, MEDIA-06)

Layout: single horizontal flex row, `justifyContent: 'center'`, `gap: 16px`, `padding: 16px`:

```
[ MuteButton ] [ CamToggleButton ] [ HangUpButton ] [ DebugToggle ]
```

- `MuteButton`: toggling calls `track.enabled = !track.enabled` on the audio track; NO renegotiation; updates `callStore.micMuted`; sends `media-state` message to remote via relay (D-14)
- `CamToggleButton`: toggling calls `track.enabled = !track.enabled` on the video track; NO renegotiation; updates `callStore.camOff`; sends `media-state` message to remote via relay (D-14)
- Both buttons: immediate visual feedback on click (button style switches before server confirmation — these are local-only actions)
- `HangUpButton`: fires `call-hangup` intent; transitions to `ended { reason: 'completed' }` (or server may override to `dropped` if both hang up simultaneously)

### Remote Status Indicators (MEDIA-01)

- `RemoteMuteIndicator`: shown when `callStore.remoteMicMuted === true`; positioned `position: absolute, top: 8px, left: 8px` on the remote video container
- `RemoteCamOffOverlay`: shown when `callStore.remoteCamOff === true`; replaces remote video content with avatar; the `<video>` element stays in DOM (stream continues) but is hidden with `display: none`; the overlay sits in the same container

### PiP Self-View (MEDIA-06)

- Position: `position: absolute, bottom: 8px, right: 8px` in remote video container (carry-forward from Phase 3)
- Cam-off state: when `callStore.camOff === true`, show a small avatar circle inside the PiP box instead of video; same 160×120 dimensions, `background: #1f2028`, centered initial letter at `fontSize: 20px`
- PiP does NOT have drag-to-reposition in Phase 4 (deferred to Phase 8 as a nice-to-have)

### Reconnect Overlay (STAB-01, STAB-02)

- Trigger: `callState === 'reconnecting'`
- Renders on top of everything in `/call` route (z-index > all call elements)
- Remote video opacity stays at 0.6 (Phase 3 behavior — last frame frozen)
- Audio muted: `remoteRef.current.volume = 0` while overlay visible; restore on `callState === 'connected'`
- Grace period (15s, env-configurable): if `callState` returns to `'connected'` within grace → overlay dismisses automatically; if grace expires → server sends `call-state-changed { reason: 'dropped' }` → transition to `ended { reason: 'dropped' }`

### Call Summary Screen (CALL-07, D-08)

- Trigger: `callState === 'ended'` (any reason)
- Renders as full-page overlay (replaces `/call` content, does NOT unmount video elements immediately)
- Auto-redirect: `setTimeout(() => navigate('/'), 3000)` — cleared by "Về ngay" click
- `dropped` variant: heading in `#dc2626`, no duration shown (call was abnormal)
- `completed` variant: shows `callDurationSec` formatted as `MM:SS` or `H:MM:SS`

### Toast System (CALL-05 busy, CALL-04 missed-callee)

- No toast library — implement with a simple Zustand-managed `toastMessage: string | null` + `setTimeout` auto-dismiss after 4000ms
- Visual: fixed bottom-center banner, `position: fixed, bottom: 24px, left: 50%, transform: translateX(-50%)`, `background: var(--code-bg)`, `borderRadius: 8px`, `padding: 12px 24px`, `boxShadow: var(--shadow)`, `fontSize: 16px`, `color: var(--text-h)`, `zIndex: 9999`
- Accessible: `role="status"` and `aria-live="polite"` on the toast element

## Ringtone Asset

- Path: `frontend/public/ringtone.mp3`
- Duration: 1-2 seconds, looping (browser `loop` attribute handles repetition)
- Executor must source a royalty-free ringtone. Acceptable fallback: generate a simple tone using the Web Audio API if no asset is available, but a pre-recorded MP3 is preferred for consistent behavior.
- Volume: default browser volume (no JavaScript volume control for ringtone)
- Stop conditions: user accepts, user rejects, server fires `missed`, server fires `busy` (busy never triggers ring on callee side)

## Accessibility Notes

- All call-action buttons have `aria-label` in Vietnamese (carry-forward pattern from Phase 3)
- `MuteButton` must update `aria-label` dynamically: "Tắt mic" when active, "Bật mic" when muted
- `CamToggleButton` must update `aria-label` dynamically: "Tắt camera" when on, "Bật camera" when off
- `ReconnectOverlay` must have `role="status"` and `aria-live="assertive"` (state change is time-sensitive)
- `CallSummaryScreen` must have `role="dialog"` and `aria-modal="true"` with `aria-labelledby` pointing to heading
- Toast container must have `role="status"` and `aria-live="polite"`
- `RemoteMuteIndicator` and `RemoteCamOffOverlay` should have `aria-label` describing the remote participant's state for screen readers
