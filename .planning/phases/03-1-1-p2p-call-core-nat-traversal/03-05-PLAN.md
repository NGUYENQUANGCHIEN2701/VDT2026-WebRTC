<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → coturn (TURN relay) | Browser presents an ephemeral HMAC credential; coturn validates against the shared secret before relaying |
| second LAN device → HTTPS/WSS endpoint | TLS terminates at Vite/Spring; mkcert CA establishes trust; getUserMedia secure-context satisfied |
| operator → .env (TURN_SECRET) | Secret is operator-supplied, server-side only; never committed, never shipped to the browser statically |
| operator → keystore (backend.p12 / *.pem) | Cert/keystore files are operator-provided via mkcert, gitignored, loaded only under the dev profile |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Information Disclosure | TURN static secret exposure | mitigate | static-auth-secret stays in turnserver.conf/.env (server-side); browser only gets the time-limited HMAC credential from /api/turn-credentials; .env.example holds placeholders only |
| T-03-04 | Elevation of Privilege | coturn used as an open relay | mitigate | use-auth-secret requires a valid HMAC; coturn rejects unauthenticated relay; fingerprint enabled |
| T-03-12 | Tampering / MITM | HTTP getUserMedia + ws:// mixed content | mitigate | HTTPS/WSS via mkcert; allowed-origins restricted to https origins (no wildcard); VITE_WS_URL uses wss:// |
| T-03-13 | Information Disclosure | committed secret/keystore in repo | mitigate | TURN_SECRET/keystore password are env-driven with placeholder examples; .p12/.pem are operator-provided + gitignored; no real secret in committed files |
| T-03-SC | Tampering | vite-plugin-mkcert install (optional) | mitigate | blocking-human legitimacy checkpoint (Task 1) before any install; RESEARCH audit = APPROVED; Approach A avoids the package entirely |
</threat_model>
