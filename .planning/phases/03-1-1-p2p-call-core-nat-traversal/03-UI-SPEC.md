# Phase 3 вҖ” UI Design Contract

> Visual and interaction contract for Phase 3: 1-1 P2P Call Core & NAT Traversal.
> Generated by gsd-ui-researcher.
>
> **Scope (new screens + augmented Home surface):**
> 1. A new `/call` route (React Router, ProtectedRoute) вҖ” the full in-call screen (D-05).
> 2. Pre-call self-view: mirrored local video preview + getUserMedia error states + audio-only fallback (D-06 / MEDIA-02 / MEDIA-05).
> 3. Call initiation from the Home online list: outgoing-call overlay, incoming-call card with Nhбәӯn/Tб»« chб»‘i, Hб»§y while calling (D-01).
> 4. In-call layout: remote video (main), local PiP self-view, hang-up control, quality indicator (RTT/packet loss), debug panel (hidden by default, toggled by button) (D-05 / STAB-03 / STAB-04).
> 5. Connection/ICE state surfacing: connecting вҶ’ connected вҶ’ reconnecting вҶ’ failed, reusing Phase 2 ConnectionIndicator visual language.
>
> **Out of scope for this UI-SPEC (вҶ’ Phase 4):** full lifecycle polish, mute/cam-off toggles, device selection dropdowns, PiP drag, call duration timer, end-reason screens, ringtone, timeoutвҶ’missed, busy, glare.

## Spacing Scale

Inherits Phase 2 scale without exception. New call-specific usages noted in the Usage column.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between status dot and label; debug panel row label-to-value gap |
| sm | 8px | Control button inner padding (vertical); gap between controls in the call toolbar |
| md | 16px | Call toolbar horizontal padding; MediaErrorNotice card inner padding; QualityIndicator internal padding |
| lg | 24px | CallPage safe-area padding from screen edge; IncomingCallCard padding |

| 3xl | 64px | Not introduced in Phase 3 UI (reserved) |

Exceptions:
- Self-view PiP: **160px Г— 120px** (fixed dimension; aspect ratio 4:3; overrides normal token grid вҖ” this is a media constraint, not layout spacing).
- Hang-up button diameter: **56px** (fixed circular touch target; exceeds `xl` for finger reachability on mobile demo).
- Control icon buttons (debug toggle, audio indicator): **44px Г— 44px** minimum touch target (WCAG 2.5.5 AAA guidance for touch).
- Remote video occupies all remaining viewport height above the control bar вҖ” no fixed token; uses `flex: 1` / `height: 100%` within the CallPage flex column.
- Video letterbox bars (black): `background: #000` вҖ” not a token, it is a media fill color (see Color).

## Interaction Patterns

### Pattern 1: Call Initiation Flow (Caller)

1. User clicks "Gб»Қi" on an `<OnlineUserRow>`.
2. `callStore` sets `callState = 'outgoing'`, `remoteUserId = targetId`.
3. `acquireMedia()` is called: tries `getUserMedia({ audio, video })`.
   - On success: `callStore` sets `localStream` reference (outside Zustand вҖ” PeerManager holds it); `callStore` sets `mediaMode = 'av'`.
   - On `NotFoundError`/`OverconstrainedError`: try audio-only fallback; if OK, `callStore` sets `mediaMode = 'audio-only'`, shows `<AudioOnlyBadge>`.
   - On other error: render `<MediaErrorNotice>`, abort call flow.
4. `<SelfViewPreview>` overlay renders with the local stream playing (mirrored).
5. WS sends `{ type: 'call-offer', to: targetId, callId }`.
6. UI shows "Дҗang gб»Қi cho {username}вҖҰ" + `<Hб»§yButton>`.
7. If callee accepts (`call-accept` arrives): navigate to `/call`; `PeerManager` starts perfect negotiation.
8. If callee rejects (`call-reject` arrives): dismiss overlay, `callStore` resets, back to normal Home.
9. If Hб»§y is tapped: WS sends `{ type: 'call-cancel', to: targetId, callId }`; dismiss overlay, `callStore` resets.

### Pattern 2: Incoming Call Flow (Callee)

1. WS receives `call-offer-received` вҶ’ `callStore` sets `callState = 'incoming'`, `remoteUserId = callerId`.
2. `<IncomingCallCard>` renders as fixed overlay on Home.
3. If "Nhбәӯn" tapped:
   a. `acquireMedia()` called (same error handling as Pattern 1).
   b. WS sends `{ type: 'call-accept', to: callerId, callId }`.
   c. Navigate to `/call`.
   d. `PeerManager` initialises with `polite = true` (callee is always polite in Phase 3).
4. If "Tб»« chб»‘i" tapped: WS sends `{ type: 'call-reject', to: callerId, callId }`; `callStore` resets; card dismisses.

### Pattern 3: getUserMedia Error + Audio-Only Fallback

1. `acquireMedia()` catches `NotFoundError` or `OverconstrainedError`.
2. Retries `getUserMedia({ audio: { echoCancellation: true, noiseSuppression: true } })`.
   - Success: continue call flow in audio-only mode; render `<AudioOnlyBadge>` in self-view; `<MediaErrorNotice>` with "Tiбәҝp tб»Ҙc vб»ӣi Гўm thanh" fallback button is NOT shown (fallback already taken automatically).
   - Failure: render `<MediaErrorNotice>` with the appropriate error copy; no call attempt proceeds.
3. For `NotAllowedError`, `NotReadableError`, `SecurityError`, `unknown`: do NOT attempt fallback; render `<MediaErrorNotice>` immediately with the correct message from the Copywriting Contract.

### Pattern 4: Debug Panel Toggle

1. Debug toggle button in the CallPage top bar (always visible in-call, `aria-expanded="false"` initially).
2. Tap/click: `DebugPanel` renders below the control bar; button `aria-expanded="true"`, border changes to `var(--accent-border)`.
3. Tap/click again: `DebugPanel` hidden; button returns to default style; `aria-expanded="false"`.
4. `DebugPanel` polls `getStats()` every **1000ms** while visible; stops polling when hidden (no wasted work).
5. The debug state (open/closed) is local component state (`useState`) вҖ” not in `callStore` (it is a developer tool, not application state).

### Pattern 5: Hang Up (either side)

1. HangUpButton tapped in-call (or `hang-up` message received from remote).
2. `PeerManager.close()` called вҖ” closes `RTCPeerConnection`, stops local media tracks.
3. WS sends `{ type: 'hang-up', to: remoteUserId, callId }` (only if locally initiated).
4. Navigate back to `/` (Home).
5. `callStore` resets to idle state.
6. No end-reason screen in Phase 3 (вҶ’ Phase 4).

### Pattern 6: ICE / Connection State Surfacing

`RTCPeerConnection` fires `oniceconnectionstatechange`. State вҶ’ UI mapping:

| `iceConnectionState` | `callStore` state | `QualityIndicator` display |
|---------------------|-------------------|---------------------------|
| `'new'` / `'checking'` | `'connecting'` | "Дҗang kбәҝt nб»‘iвҖҰ" grey pulsing dot |
| `'connected'` / `'completed'` | `'connected'` | "ДҗГЈ kбәҝt nб»‘i" green/accent dot + RTT/Loss |
| `'disconnected'` | `'reconnecting'` | "Дҗang kбәҝt nб»‘i lбәЎiвҖҰ" destructive dot; remote video dims |
| `'failed'` | `'failed'` | "Kбәҝt nб»‘i thбәҘt bбәЎi" destructive dot |
| `'closed'` | `'idle'` | (navigate away вҖ” this state is never displayed) |

**Important:** `callStore` only holds the derived string state (`'connecting' | 'connected' | 'reconnecting' | 'failed' | 'idle'`), not the `RTCPeerConnection` object itself (CLAUDE.md mandate: non-serializable objects stay in PeerManager, not Zustand).

## Accessibility Baseline

- `<QualityIndicator>`: `role="status" aria-live="polite"` вҖ” ICE state changes announced (mirrors Phase 2 `ConnectionIndicator`).
- `<MediaErrorNotice>`: `role="alert"` вҖ” error is announced immediately on render.
- `<IncomingCallCard>` overlay: `role="dialog" aria-modal="true" aria-labelledby="incoming-call-heading"`. Focus is trapped inside the card (Nhбәӯn / Tб»« chб»‘i are the only interactive elements). On dismiss, focus returns to the element that had focus before the card appeared.
- `<HangUpButton>`: `aria-label="Kбәҝt thГәc cuб»ҷc gб»Қi"` (icon-only button must have accessible label).
- `<DebugToggleButton>`: `aria-label="BбәЈng debug (бә©n/hiб»Үn)" aria-expanded={open}`.
- `<NhбәӯnButton>`: `aria-label="Nhбәӯn cuб»ҷc gб»Қi"`. `<Tб»«ChoiButton>`: `aria-label="Tб»« chб»‘i cuб»ҷc gб»Қi"`.
- Self-view `<video>`: `aria-label="Camera cб»§a bбәЎn" aria-hidden="false"`. Remote `<video>`: `aria-label="Camera cб»§a {remoteUsername}"`.
- `<AudioOnlyBadge>`: text "Chб»ү Гўm thanh" вҖ” visible and in the DOM; not `aria-hidden`.
- Status conveyed by colour + text, never colour alone (ICE dot = decorative; text carries meaning).
- Colour contrast: all new colour pairings meet WCAG 2.1 AA (see Color section contrast notes).
- Keyboard: all call buttons reachable via Tab; HangUpButton is the last focusable element in the call control bar (natural Tab order).
