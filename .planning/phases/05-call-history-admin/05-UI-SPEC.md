# Phase 5 вҖ” UI Design Contract

> Visual and interaction contract for the three Phase 5 UI surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Surfaces in this phase:**
> 1. **User call-history view** вҖ” per-day grouped call log, infinite scroll, read-only (HIST-02, D-01..D-07).
> 2. **Admin user management** вҖ” extend existing `AdminPage` users table with inline lock/unlock + role dropdown + confirmation + self-protection (ADMN-01, D-08..D-11).
> 3. **Admin live dashboard + system-wide history** вҖ” stat cards (~5s poll) + system call table with username filter (ADMN-02/03, D-12..D-16).
>
> **Established language (binding):** this app uses **plain inline styles + CSS custom properties** (`frontend/src/index.css`), NOT Tailwind and NOT a component library. This contract codifies the tokens already in use across `CallPage`, `components/call/*`, `components/presence/*`, and `Toaster`. Phase 5 MUST match that language вҖ” do not introduce Tailwind, shadcn, or a UI kit.

## Typography

Sizes and weights taken from `index.css` and existing components вҖ” exactly **4 sizes** and **3 weights** in active use; Phase 5 stays within them.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display / page title (`h1`) | 56px (36px вүӨ1024px via existing `h1` rule) | 500 | ~1.1 (`letter-spacing: -1.68px`, existing) |

**Phase-5 specifics:**
- **Stat-card big number:** 40вҖ“48px, weight 600, `var(--text-h)`, `fontVariantNumeric: 'tabular-nums'`. This is the one place a larger numeric display is justified ("read clearly during a live demo", D-16) вҖ” reuse the `h1`/display scale rather than inventing a new size; specify 44px.
- **Stat-card label:** 14px, weight 600, `var(--text)` (muted), uppercase optional.
- **Day-group header (history):** 14px, weight 600, `var(--text)` muted, with bottom border `1px solid var(--border)` вҖ” matches table-header treatment.
- **Duration column:** 16px, `tabular-nums`, so digits align in the list.

> Note: the legacy root size is 18px and `h1` is 56px. These are pre-existing global rules. Phase 5 does not change globals; it works within them. The 4-row table above is the *contract for Phase-5-authored elements*.

## Color

The palette is light/dark via `prefers-color-scheme` using CSS variables. **Always reference the variable, never hardcode the surface hex** (so dark mode keeps working). Semantic status colors follow the established hardcoded pairs already used in `StatusBadge`, `Toaster`, `OnlineUserRow`, and `CallSummaryScreen`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` (`#fff` light / `#16171d` dark) | Page background, table body rows |
| Secondary (30%) | `var(--code-bg)` (`#f4f3ec` light / `#1f2028` dark) + `var(--border)` for table-header band / stat-card surface / modal card | Stat cards, table header row, day-group dividers, confirmation modal card, hover row |
| Accent (10%) | `var(--accent)` (`#aa3bff` light / `#c084fc` dark) | See reserved-for list below |
| Destructive | `#dc2626` text on `#fee2e2` bg (red) | Lock action button + confirm button; "Cuб»ҷc gб»Қi nhб»Ў" / missed direction glyph |

**Accent (`var(--accent)`) reserved for вҖ” explicit list (no other element may use accent):**
1. The primary confirm/CTA button in the lock/role-change confirmation modal ("XГЎc nhбәӯn") вҖ” solid accent fill, white text (matches `CallSummaryScreen` "Vб»Ғ ngay").
2. Active/selected pagination or "load more" affordance focus ring (`var(--accent-border)`).
3. The username-filter input focus border in the admin system-history view (`var(--accent-border)`), matching the `OnlineUserRow` "Gб»Қi" button outline convention.

Accent is **not** used for: table headers, links in cells, stat-card numbers, role dropdowns, or generic interactive rows.

**Semantic status palette (reuse exactly вҖ” established):**

| Meaning | Text | Background | Source |
|---------|------|------------|--------|
| Completed / success / "HoбәЎt Д‘б»ҷng" (unlocked) | `#166534` | `#dcfce7` | `StatusBadge` ONLINE |
| In-progress / warning / "Дҗang gб»Қi" / dropped | `#92400e` (badge) / `#d97706` (text) | `#fef3c7` | `StatusBadge` IN_CALL, `Toaster` warning, `CallSummaryScreen` dropped |
| Missed / locked / destructive | `#dc2626` | `#fee2e2` | new pair, consistent with red destructive convention |
| Neutral / completed outcome label | `var(--text)` | transparent / `var(--code-bg)` | `CallSummaryScreen` neutral |

**Outcome-label color mapping (history rows, per-side, D-06):**
- `completed` вҶ’ neutral (`var(--text)`).
- `missed` (callee view "Cuб»ҷc gб»Қi nhб»Ў") / `cancelled`вҶ’missed-side вҶ’ red (`#dc2626`).
- `rejected` вҶ’ neutral/muted.
- `dropped` вҶ’ amber `#d97706` (warning, matches summary screen).
- Direction glyph color: incoming/outgoing neutral `var(--text)`; missed glyph red `#dc2626`.

> Note on the 60/30/10 split: this is an information-dense admin/data product (tables + cards), so "accent 10%" is interpreted as *accent restraint* (CTAs and focus only). The dominant/secondary surfaces carry the layout; semantic status colors are small-area pills/glyphs and do not count against the accent budget.

## Component / Interaction Inventory (Phase 5)

Prescriptive notes for the executor; all built as plain inline-styled elements.

**Surface 1 вҖ” User call-history (`/history` or HomePage section):**
- Container: max-width 800, centered, `padding: 24px`.
- `useInfiniteQuery` (D-03) with an IntersectionObserver sentinel at list bottom; fetch next page on intersect. Show `Дҗang tбәЈiвҖҰ` row while fetching next page.
- Day group: sticky 14px/600 muted header with bottom border; rows newestвҶ’oldest within group (D-02).
- Row layout (flex, `padding: 12px 16px`, `borderBottom: 1px solid var(--border)`): `[direction glyph] [peer username (16/600, var(--text-h), flex:1, text-align:left)] [outcome label (14, semantic color)] [duration (16, tabular-nums)] [timestamp (14, muted)]`. Read-only вҖ” no row actions, no delete (D-04).
- Per-side perspective rendering driven by viewer identity (D-06).

**Surface 2 вҖ” Admin user management (extend `AdminPage` table):**
- Keep existing columns (ID, Username, Email, Role, TrбәЎng thГЎi); ADD an `HГ nh Д‘б»ҷng` column.
- Role cell вҶ’ `<select>` (Admin/User), 14px, disabled on own row (D-10). On change вҶ’ confirmation modal before committing.
- Action cell вҶ’ `KhГіa`/`Mб»ҹ khГіa` button. Lock = destructive style (red text `#dc2626`, border `#dc2626` or fill on confirm). Unlock = neutral/accent outline (matches `OnlineUserRow` "Gб»Қi" button: transparent bg, colored border, radius 4, padding `4px 8px`, 14/600). Hidden/disabled on own row (D-10).
- After a successful lock the row's TrбәЎng thГЎi flips to `ДҗГЈ khГіa`; no page reload (optimistic or refetch).

**Surface 3 вҖ” Admin dashboard + system history:**
- Dashboard: responsive row of 5 stat cards (`var(--code-bg)` surface, radius 12, padding 24, `var(--shadow)`, gap 16). Each: label (14/600 muted) above number (44/600 `var(--text-h)`, tabular-nums). Poll via TanStack Query `refetchInterval: 5000` (D-15); keep prior values on transient error.
- System history: same table/row visual language as Surface 1 but shows BOTH parties (caller вҶ’ callee) per row (D-12), newestвҶ’oldest, paginated. Username filter input above the table (placeholder `Lб»Қc theo tГӘn ngЖ°б»қi dГ№ngвҖҰ`, focus border `var(--accent-border)`).

## Accessibility & States Checklist (binding)

- Confirmation modal: `role="dialog" aria-modal="true" aria-labelledby` (mirror `CallSummaryScreen`). Focus the confirm button on open; Esc / overlay click cancels.
- Direction glyphs are decorative (`aria-hidden="true"`); the textual outcome/direction label carries meaning (mirror the `OnlineUserRow` dot + `StatusBadge` pattern).
- Disabled own-row controls use `disabled` + `opacity: 0.4` + `cursor: not-allowed` (matches `OnlineUserRow` disabled "Gб»Қi").
- Every surface renders loading, empty, and error states from the copy table above вҖ” never a blank screen.
- Duration and stat numbers use `tabular-nums` for column alignment.
