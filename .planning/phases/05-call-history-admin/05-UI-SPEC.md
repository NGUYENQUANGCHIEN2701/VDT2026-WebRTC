# Phase 5 ‚Äî UI Design Contract

> Visual and interaction contract for the three Phase 5 UI surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Surfaces in this phase:**
> 1. **User call-history view** ‚Äî per-day grouped call log, infinite scroll, read-only (HIST-02, D-01..D-07).
> 2. **Admin user management** ‚Äî extend existing `AdminPage` users table with inline lock/unlock + role dropdown + confirmation + self-protection (ADMN-01, D-08..D-11).
> 3. **Admin live dashboard + system-wide history** ‚Äî stat cards (~5s poll) + system call table with username filter (ADMN-02/03, D-12..D-16).
>
> **Established language (binding):** this app uses **plain inline styles + CSS custom properties** (`frontend/src/index.css`), NOT Tailwind and NOT a component library. This contract codifies the tokens already in use across `CallPage`, `components/call/*`, `components/presence/*`, and `Toaster`. Phase 5 MUST match that language ‚Äî do not introduce Tailwind, shadcn, or a UI kit.

## Typography

**Exactly two weights** for all Phase-5-authored elements: **400 (regular)** for body text, and **600 (semibold)** for every heading, page title, label, button, badge, emphasis, stat number, and modal title. No third weight (e.g. 500) is used. This is a Phase-5 contract decision only ‚Äî the global `index.css` `h1/h2 { font-weight: 500 }` rule is NOT edited; Phase-5-authored headings/titles render at 600.

Four sizes, two weights:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display / page title | 56px (36px ‚â§1024px via existing `h1` rule) | 600 | ~1.1 (`letter-spacing: -1.68px`, existing) |
| Heading / section / modal title | 22‚Äì24px | 600 | 1.18 |
| Body / table cell / username | 16px | 400 (regular); 600 only for emphasis (username, stat number) | 1.45 (root `font: 18px/145%`; cells 16px) |
| Label / badge / button / meta | 14px | 600 | 1.0‚Äì1.2 (badges use `lineHeight: 1`) |

**Phase-5 specifics:**
- **Stat-card big number:** 44px, weight 600, `var(--text-h)`, `fontVariantNumeric: 'tabular-nums'`. This is the one place a larger numeric display is justified ("read clearly during a live demo", D-16) ‚Äî reuse the display scale rather than inventing a new size.
- **Stat-card label:** 14px, weight 600, `var(--text)` (muted), uppercase optional.
- **Day-group header (history):** 14px, weight 600, `var(--text)` muted, with bottom border `1px solid var(--border)` ‚Äî matches table-header treatment.
- **Duration column:** 16px, weight 400, `tabular-nums`, so digits align in the list.

> Note: the legacy root size is 18px and the global `h1` is 56px/500. These are pre-existing global rules. Phase 5 does not change globals; Phase-5-authored elements follow the 2-weight (400/600) contract above.

## Color

The palette is light/dark via `prefers-color-scheme` using CSS variables. **Always reference the variable, never hardcode the surface hex** (so dark mode keeps working). Semantic status colors follow the established hardcoded pairs already used in `StatusBadge`, `Toaster`, `OnlineUserRow`, and `CallSummaryScreen`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` (`#fff` light / `#16171d` dark) | Page background, table body rows |
| Secondary (30%) | `var(--code-bg)` (`#f4f3ec` light / `#1f2028` dark) + `var(--border)` for table-header band / stat-card surface / modal card | Stat cards, table header row, day-group dividers, confirmation modal card, hover row |
| Accent (10%) | `var(--accent)` (`#aa3bff` light / `#c084fc` dark) | See reserved-for list below |
| Destructive | `#dc2626` text on `#fee2e2` bg (red) | Lock action button + confirm button; "Cu·ªôc g·ªçi nh·ª°" / missed direction glyph |

**Accent (`var(--accent)`) reserved for ‚Äî explicit list (no other element may use accent):**
1. The primary confirm/CTA button in the lock/role-change confirmation modal ("X√°c nh·∫≠n") ‚Äî solid accent fill, white text (matches `CallSummaryScreen` "V·ªÅ ngay").
2. Active/selected pagination or "load more" affordance focus ring (`var(--accent-border)`).
3. The username-filter input focus border in the admin system-history view (`var(--accent-border)`), matching the `OnlineUserRow` "G·ªçi" button outline convention.

Accent is **not** used for: table headers, links in cells, stat-card numbers, role dropdowns, or generic interactive rows.

**Semantic status palette (reuse exactly ‚Äî established):**

| Meaning | Text | Background | Source |
|---------|------|------------|--------|
| Completed / success / "Ho·∫°t ƒë·ªông" (unlocked) | `#166534` | `#dcfce7` | `StatusBadge` ONLINE |
| In-progress / warning / "ƒêang g·ªçi" / dropped | `#92400e` (badge) / `#d97706` (text) | `#fef3c7` | `StatusBadge` IN_CALL, `Toaster` warning, `CallSummaryScreen` dropped |
| Missed / locked / destructive | `#dc2626` | `#fee2e2` | new pair, consistent with red destructive convention |
| Neutral / completed outcome label | `var(--text)` | transparent / `var(--code-bg)` | `CallSummaryScreen` neutral |

**Outcome-label color mapping (history rows, per-side, D-06):**
- `completed` ‚Üí neutral (`var(--text)`).
- `missed` (callee view "Cu·ªôc g·ªçi nh·ª°") / `cancelled`‚Üímissed-side ‚Üí red (`#dc2626`).
- `rejected` ‚Üí neutral/muted.
- `dropped` ‚Üí amber `#d97706` (warning, matches summary screen).
- Direction glyph color: incoming/outgoing neutral `var(--text)`; missed glyph red `#dc2626`.

> Note on the 60/30/10 split: this is an information-dense admin/data product (tables + cards), so "accent 10%" is interpreted as *accent restraint* (CTAs and focus only). The dominant/secondary surfaces carry the layout; semantic status colors are small-area pills/glyphs and do not count against the accent budget.

## Component / Interaction Inventory (Phase 5)

Prescriptive notes for the executor; all built as plain inline-styled elements.

**Surface 1 ‚Äî User call-history (`/history` or HomePage section):**
- Container: max-width 800, centered, `padding: 24px`.
- `useInfiniteQuery` (D-03) with an IntersectionObserver sentinel at list bottom; fetch next page on intersect. Show `ƒêang t·∫£i‚Ä¶` row while fetching next page.
- Day group: sticky 14px/600 muted header with bottom border; rows newest‚Üíoldest within group (D-02).
- Row layout (flex, `padding: 12px 16px`, `borderBottom: 1px solid var(--border)`): `[direction glyph] [peer username (16/600, var(--text-h), flex:1, text-align:left)] [outcome label (14, semantic color)] [duration (16/400, tabular-nums)] [timestamp (14, muted)]`. Read-only ‚Äî no row actions, no delete (D-04).
- Per-side perspective rendering driven by viewer identity (D-06).

**Surface 2 ‚Äî Admin user management (extend `AdminPage` table):**
- Keep existing columns (ID, Username, Email, Role, Tr·∫°ng th√°i); ADD an `H√Ýnh ƒë·ªông` column.
- Role cell ‚Üí `<select>` (Admin/User), 14px, disabled on own row (D-10). On change ‚Üí confirmation modal before committing.
- Action cell ‚Üí `Kh√≥a`/`M·ªü kh√≥a` button. Lock = destructive style (red text `#dc2626`, border `#dc2626` or fill on confirm). Unlock = neutral/accent outline (matches `OnlineUserRow` "G·ªçi" button: transparent bg, colored border, radius 4, padding `4px 8px`, 14/600). Hidden/disabled on own row (D-10).
- After a successful lock the row's Tr·∫°ng th√°i flips to `ƒê√£ kh√≥a`; no page reload (optimistic or refetch).

**Surface 3 ‚Äî Admin dashboard + system history:**
- Dashboard: responsive row of 5 stat cards (`var(--code-bg)` surface, radius 12, padding 24, `var(--shadow)`, gap 16). Each: label (14/600 muted) above number (44/600 `var(--text-h)`, tabular-nums). Poll via TanStack Query `refetchInterval: 5000` (D-15); keep prior values on transient error.
- System history: same table/row visual language as Surface 1 but shows BOTH parties (caller ‚Üí callee) per row (D-12), newest‚Üíoldest, paginated. Username filter input above the table (placeholder `L·ªçc theo t√™n ng∆∞·ªùi d√πng‚Ä¶`, focus border `var(--accent-border)`).

## Accessibility & States Checklist (binding)

- Confirmation modal: `role="dialog" aria-modal="true" aria-labelledby` (mirror `CallSummaryScreen`). Focus the confirm button on open; Esc / overlay click cancels.
- Direction glyphs are decorative (`aria-hidden="true"`); the textual outcome/direction label carries meaning (mirror the `OnlineUserRow` dot + `StatusBadge` pattern).
- Disabled own-row controls use `disabled` + `opacity: 0.4` + `cursor: not-allowed` (matches `OnlineUserRow` disabled "G·ªçi").
- Every surface renders loading, empty, and error states from the copy table above ‚Äî never a blank screen.
- Duration and stat numbers use `tabular-nums` for column alignment.
