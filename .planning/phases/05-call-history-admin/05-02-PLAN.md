<objective>
## Phase Goal

**As a** logged-in user, **I want to** see my call history after every call ends, **so that** I have a durable record of all my calls with direction, duration, and outcome — and every call is recorded asynchronously without touching the realtime path.

Vertical slice 1 of 3: async history pipeline + user history UI.

This plan wires the complete end-to-end path: CallService terminal transition → fire-and-forget
publish to RabbitMQ → consumer persists two rows per call → GET /api/history returns per-side
paginated view → HistoryPage renders day-grouped call log.

Covers all three HIST requirements (D-01..D-07).

Purpose: The realtime call path (WS broadcast) must never wait on persistence. This slice
establishes the fire-and-forget AMQP pipeline and makes the database write invisible to call
quality. Users gain durable history; the system gains a durable audit trail.

Output: RabbitMQ infra in docker-compose/yaml/pom, Flyway V3 migration, all history domain
classes, CallService wired with publisher at 5 terminal points, HistoryController, and a working
/history page.
</objective>
