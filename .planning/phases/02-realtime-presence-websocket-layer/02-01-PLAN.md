    - `PresenceSweeperTest` (PRES-02 / D-04): one client connects then STOPS sending pings; using Awaitility,
      assert it is evicted from a subsequent snapshot within the TTL+sweep window (assert within ~80s ceiling;
      see RESEARCH Open Question #2 — the implementation may tighten the sweep). Keep the await timeout
      generous but bounded.
    - `SingleSessionTest` (PRES-03 / T-02-02): two clients connect with the SAME user's token; assert the
      FIRST receives a `{type:session-superseded}` frame and its session then closes.
