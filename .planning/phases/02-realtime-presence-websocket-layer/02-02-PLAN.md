<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /ws (HTTP upgrade, `?token=` query param) | Untrusted client asserts a JWT; the handshake interceptor is the sole gate that establishes server-owned identity. |
| client message body → handler | Untrusted JSON frames; only the sealed `ClientMessage` envelope is accepted; any `from` field in the body is ignored. |
| concurrent I/O threads + scheduler thread → sessions map / sendMessage | Internal concurrency boundary; corruption risk if unsynchronized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing / Elevation | JwtHandshakeInterceptor (unauthenticated WS upgrade) | mitigate | Validate JWT via existing `JwtService.isTokenValid`; reject (401, return false) on null/invalid/expired. `/ws/**` permitAll only lets the GET reach the interceptor — the interceptor is the gate. Covered by `WsHandshakeAuthTest`. |
| T-02-02 | Spoofing / Tampering | PresenceWebSocketHandler identity binding | mitigate | Identity read ONLY from `session.getAttributes().get("username")` (stamped at handshake); body `from` never read. Covered by `WsIdentityTest`. |
| T-02-03 | Information Disclosure | Token in `?token=` query string (logs) | accept | Short-lived (15-30min) access token limits blast radius; never log the token (`log.warn` without credential, Security V7). Documented known risk; v2 one-time-ticket hardening (STAB-05) deferred. LB access-log suppression handled in Phase 6. |
| T-02-04 | Tampering | sessions map supersede/reconnect race | mitigate | `ConcurrentHashMap` + `sessions.remove(user, session)` value-checked removal + session-id compare before kick (Pitfall 2). Covered by `SingleSessionTest`. |
| T-02-05 | Tampering / Availability | concurrent `sendMessage` on one session | mitigate | `synchronized(session)` around every send with `isOpen()` guard (Pitfall 3). |
| T-02-06 | Spoofing (CSWSH) | cross-site WebSocket hijack | mitigate | `setAllowedOrigins("http://localhost:5173")` (explicit frontend origin, never `*`) matching CorsConfig (Pitfall 6). |
| T-02-07 | Denial of Service | reconnect storm / connection flood | accept | Client backoff+jitter (Plan 03); per-user connection cap + send-buffer limit noted for Phase 6 scale hardening. Low risk at demo scale. |
| T-02-SC | Tampering | Maven install (spring-boot-starter-websocket) | accept | First-party Spring Boot starter, BOM-managed by the locked parent 4.0.7; RESEARCH Package Legitimacy Audit verdict Approved. No [ASSUMED]/[SUS] package — no blocking checkpoint required. |
</threat_model>
