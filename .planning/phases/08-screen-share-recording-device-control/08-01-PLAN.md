## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSocket client → backend recording relay | Tests must assert server validates call participation before relaying; arbitrary clients cannot inject recording indicators for calls they are not in. |
| Frontend media modules → Zustand | Tests must not assert MediaStream, MediaRecorder, AudioContext, or RTCPeerConnection ever appear in Zustand state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | recording-state relay | mitigate | RED tests require areActiveCallPeers validation before relay proceeds; backend cannot relay to arbitrary users. |
| T-08-02 | Tampering | recording indicator | mitigate | Tests assert relay is rejected for non-participants, preventing a rogue client from setting a false recording flag on a call they are not in. |
| T-08-03 | Information Disclosure | RecordingState callId | mitigate | Relay test validates actorId membership in callId before relaying callId to recipient, so callId is not leaked to non-participants. |
</threat_model>
