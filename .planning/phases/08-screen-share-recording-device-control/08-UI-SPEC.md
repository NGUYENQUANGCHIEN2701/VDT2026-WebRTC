# Phase 08 - UI Design Contract

> Visual and interaction contract for screen share, recording, and device control. Generated by gsd-ui-researcher generic-agent workaround.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none; manual React + CSS |
| Preset | not applicable |
| Component library | none |
| Icon library | lucide-react |
| Font | Inter, system-ui, Segoe UI, Roboto, sans-serif |

Existing call UI is the source of truth. New UI must extend `CallPage.tsx`, `GroupCallPage.tsx`, `CallButtons.tsx`, `ParticipantTile.tsx`, and `index.css` patterns instead of introducing a new framework.

| Token | Value | Usage |

## Screen Share Contract

### 1-1 Calls

- `LabeledShareButton` in `CallPage.tsx` becomes the single screen-share toggle.
- Default state: MonitorUp icon, label `Share` or localized equivalent, title and `aria-label` `Share screen`.
- Active state: use `call-labeled-btn active` styling; icon surface background `#3b82f6`; label `Sharing`; title and `aria-label` `Stop sharing`.
- While requesting display capture, disable the share button and expose `aria-busy="true"` with label `Starting...`.
- Starting screen share explicitly turns video sharing on for the screen track even if camera was off. The camera-off visual state must clear while sharing, and media-state must be relayed.
- Stopping from the app or browser toolbar must restore the current camera track automatically. If the user turned camera off while sharing, stopping must restore camera-off state and show the avatar/placeholder.
- If screen-share start fails, keep the camera track and previous UI state unchanged; show a warning toast.
- Remote users see the shared screen in the existing remote video surface; there is no extra tile or second stream in this phase.

### Group Calls

- `LabeledShareButton` in `GroupCallPage.tsx` becomes the room screen-share toggle with the same inactive, loading, active, and error states.
- Screen share fans out to every peer. Every remote participant sees the sharer's screen inside that participant's existing `ParticipantTile`.
- A participant who joins after the sharer switched to screen share receives the current shared track.
- Stopping from browser toolbar restores the camera track across all peers and updates the sharer's local tile.
- Group recording controls must not appear.

### Active Sharing Indicator

- Add a compact HUD pill near the top-center call status: MonitorUp icon + `Sharing screen`.
- In group calls, remote participant tiles showing a screen share get a small top-left badge `Screen` unless it conflicts with mute; if both exist, stack badges with 8px gap.
- Active share badges use blue accent, not red, because sharing is not an alarm state.

## More Panel and Device Selectors

### Placement and Trigger

- 1-1: `LabeledMoreButton` opens a More panel anchored above the bottom bar, centered horizontally with the controls.
- Group: add `LabeledMoreButton` to the bottom bar or wire the top-right `MoreVertical`; the bottom bar must still keep Participants reachable.
- Panel width: 360px desktop, `calc(100vw - 32px)` mobile. Max height: `min(70dvh, 520px)` with internal scrolling.
- Panel surface: same dark glass language as call HUDs, 16px radius, 1px translucent border, 16px padding.
- Dismiss on Escape, outside click, Hang Up/Leave, and successful route away. Focus returns to the More button.

### Required Controls

Camera selector:
- Label `Camera`.
- Shows current selected camera label, or `Camera 1` fallback when labels are blank.
- Disabled while switching camera.
- If camera is off, selector remains enabled and note text says the new camera will stay off until Camera is turned on.
- If screen sharing, switching camera updates the saved camera selection but does not interrupt screen share; note text says it applies after sharing stops.

Microphone selector:
- Label `Microphone`.
- Shows current selected microphone label, or `Microphone 1` fallback.
- Disabled while switching microphone.
- If muted, selector remains enabled and note text says the new microphone will stay muted.

Speaker selector:
- Label `Speaker`.
- Render only when `setSinkId` exists on `HTMLMediaElement.prototype`.
- If unsupported, hide the selector entirely from the panel and show no disabled dead control.
- If setting sink fails, retain the previous output and show a warning toast.

### Device List States

- Loading: one row skeleton or muted text `Loading devices...`.
- Empty: render the empty devices copy from the copywriting table inside the panel.
- Permission pending or labels blank: show fallback labels and a small helper `Browser permission may be needed to show device names.`
- Device changed while panel is open: refresh options without closing the panel; preserve selected value where still available.

### State Preservation

- Device switching must preserve `micMuted` and `camOff`.
- Replacing tracks must not toggle the visible mute/camera buttons, remote indicators, or recording state.
- On switch failure, the previous track remains active and button/selector state rolls back.

## Recording Contract

### Scope

- Recording is available only on the 1-1 call page.
- Recording is local browser recording only; do not upload, do not add history metadata, and do not show recording controls in group calls.
- The recording output is a composited `.webm`: remote video main surface, local self-view PiP, mixed local + remote audio.

### Control Placement

- Recording controls live in the More panel under a `Recording` section.
- Idle state: button `Start recording` with a record icon/dot.
- Active state: button `Stop recording` styled destructive, plus timer `Recording 00:00`.
- If `MediaRecorder` is unsupported or no remote stream exists, keep Start disabled with clear helper text.
- Starting a recording closes the More panel only on mobile; on desktop it may stay open if it does not obscure the call.

### Local Recording Indicator

- While recording, show a persistent red top-center HUD pill: red dot + `Recording` + elapsed timer.
- The indicator must be visible even when More is closed.
- The indicator has `role="status"` and `aria-live="polite"` only when recording starts/stops, not on every timer tick.
- Hanging up while recording stops the recording, prepares preview if chunks exist, and clears remote indicator best-effort.

### Remote Recording Indicator

- When a peer records, the remote side shows a persistent red top-center or top-left HUD pill: red dot + `{username} is recording`.
- It remains until `recording-state-relay` reports false or the call ends.
- Remote indicator is informational only; it has no controls.
- If local and remote recording are both true, show both in a combined stacked/inline status area without covering the video subject.

### Stop Preview and Download

- Stopping recording opens a modal/panel above the call page with:
  - heading `Recording ready`
  - video preview with controls
  - file metadata row: duration and file type when available
  - primary CTA `Download recording`
  - secondary action `Close preview`
- Modal size: `min(100%, 720px)` width, 16px radius, dark surface, 24px padding.
- Preview video aspect ratio is 16:9 and never exceeds 60dvh.
- Download filename contract: `call-{callId}-{timestamp}.webm`.
- Closing the preview does not auto-download and does not delete the object URL until the modal is dismissed; object URL must be revoked after dismissal or replacement.

## Error and Permission States

Screen share:
- `NotAllowedError`: warning toast with permission copy.
- `NotReadableError` / `AbortError`: warning toast `Could not start screen sharing. Try another window or screen.`
- Unsupported `getDisplayMedia`: disable Share and title `Screen sharing is unavailable in this browser.`

Devices:
- `NotAllowedError`: show warning toast and keep previous device.
- `NotFoundError`: empty state in More panel.
- `OverconstrainedError`: warning toast `Selected device is unavailable. Your current device is still active.`
- `NotReadableError`: warning toast `That device is busy. Your current device is still active.`

Recording:
- Unsupported `MediaRecorder`: disabled Start with unsupported copy.
- No remote stream: disabled Start with helper `Recording starts after the call media is connected.`
- Runtime recorder error: stop recording, show destructive/error notice, keep call running.
- Empty chunks after stop: show `No recording data was captured.` with Close only.

All errors must leave the call connected and preserve the last known-good media track.

## Responsive Behavior

Desktop:
- Bottom control bar remains centered at bottom 32px.
- More panel opens above the bar and must not cover the hang-up control.
- Recording preview modal centers in the viewport over a backdrop.

Tablet and mobile (`max-width: 760px`):
- Bottom bar may use horizontal scroll or tighter gaps, but each control remains at least 44px.
- Share, More, and Hang Up must remain visible without opening another menu.
- Button text may be visually hidden below 420px, but `aria-label` and `title` remain descriptive.
- More panel becomes a bottom sheet: left/right 16px, bottom 16px, max height 72dvh.
- Recording preview modal becomes a bottom sheet with preview first and CTAs pinned below the preview.
- Top HUDs may wrap into two rows, but recording and sharing indicators must not overlap the self-view or participant labels.

Group layout:
- Participant tiles keep stable grid sizing while badges appear/disappear.
- Badge overlays must not change tile dimensions.
- In a 4-person grid, active screen-share badge and mute badge stack inside each tile without covering the participant label.

## Accessibility Contract

- All controls are native buttons/selects or equivalent elements with keyboard support.
- More panel uses dialog semantics: `role="dialog"`, `aria-modal="false"` for anchored popover or `true` for mobile sheet/modal, labelled by `More options`.
- Focus is trapped only for modal preview and mobile bottom sheet. Anchored desktop popover may use roving natural tab order.
- Escape closes More and preview modal.
- Every icon-only or icon-led control has `aria-label` matching its current action, not just its feature name.
- Active toggle buttons use `aria-pressed`: mic, camera, screen share, recording where applicable.
- Loading actions set `disabled` and `aria-busy="true"`.
- Timers use tabular numerals. Do not announce every second through live regions.
- Color is never the only state cue: badges include text, active buttons include label/title and `aria-pressed`.
- Video elements retain `playsInline`; local self-view remains muted; sink ID is never applied to local muted preview.

## Store and State UI Contract

Serializable call state may include:
- `isScreenSharing`
- `localStreamVersion`
- `selectedCameraDeviceId`
- `selectedMicrophoneDeviceId`
- `selectedSpeakerDeviceId`
- `isRecording`
- `recordingStartedAt`
- `remoteRecording`
- `recordingError`
- `hasRecordingPreview`

Serializable room state may include:
- `isScreenSharing`
- `localStreamVersion` or self member stream bump
- `selectedCameraDeviceId`
- `selectedMicrophoneDeviceId`
- `selectedSpeakerDeviceId`

Never store `MediaStream`, `MediaStreamTrack`, `MediaRecorder`, `AudioContext`, `Blob`, object URL, `RTCPeerConnection`, or `MediaDeviceInfo` in Zustand. Object URLs may live in component/service state and must be cleaned up.

## Implementation Surface Contract

Existing components:
- `CallButtons.tsx`: extend `LabeledShareButton` and `LabeledMoreButton` to accept active/loading/disabled labels and `aria-pressed`.
- `CallPage.tsx`: wire share, More, sink ID on `remoteRef`, local stream version refresh, local/remote recording indicators, preview modal.
- `GroupCallPage.tsx`: wire room share, More/device controls, sink ID for remote tiles, self tile refresh, no recording UI.
- `ParticipantTile.tsx`: accept `sinkId`, screen-share badge state, and stable badge stacking.
- `callStore.ts` and `roomStore.ts`: add serializable flags only.

Visual CSS:
- Add call-specific classes in `index.css`; avoid inline style growth for new reusable UI.
- Reuse `.call-1v1-bottom-bar`, `.call-labeled-btn`, `.modal-backdrop`, `.modal-card`, `.app-select`, and toast patterns where practical.
- New call surfaces must use 16px or smaller radii except circular status dots and pill badges.
