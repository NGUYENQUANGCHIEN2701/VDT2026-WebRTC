  <action>
    Create `frontend/src/webrtc/recording.ts` exporting a `RecordingController` class.
    This module must keep all non-serializable objects (canvas, AudioContext, MediaRecorder,
    Blob, object URL, animation frame ID) inside the class — never in Zustand.

    **MIME fallback helper** (exported for testing):
    ```ts
    export function selectMimeType(): string {
        const candidates = [
            'video/webm;codecs=vp9,opus',
            'video/webm;codecs=vp8,opus',
            'video/webm;codecs=h264,opus',
            'video/webm',
            '',
        ]
        return candidates.find(
            (m) => m === '' || MediaRecorder.isTypeSupported(m)
        ) ?? ''
    }
    ```

    **RecordingController** class:

    Constructor args: `{ callId: string, localLabel: string, remoteLabel: string }`.

    Methods:
    - `start(localStream: MediaStream, remoteStream: MediaStream): void`
    - `stop(): Promise<{ previewUrl: string; mimeType: string; durationMs: number } | null>`
    - `get isRecording(): boolean`
    - `cleanup(): void` (call on component unmount even if not recording)

    `start()` implementation:
    1. Guard: if already recording, return.
    2. Create `offscreenCanvas = document.createElement('canvas')` set to 1280×720.
    3. Create hidden `localVideo` and `remoteVideo` elements, assign streams as srcObject, call `.play()`.
    4. Create `AudioContext`, connect local and remote audio streams to `createMediaStreamDestination()`.
       Resume AudioContext (user gesture already happened by this point).
    5. Create composed stream: `new MediaStream([...canvas.captureStream(30).getVideoTracks(), ...dest.stream.getAudioTracks()])`.
    6. Select MIME via `selectMimeType()`.
    7. Create `MediaRecorder(composedStream, { mimeType: selectedMime || undefined })`.
    8. Start draw loop via `requestAnimationFrame`:
       - Draw `remoteVideo` as full canvas (or placeholder rect if `readyState < HAVE_CURRENT_DATA`).
       - Draw `localVideo` as bottom-right PiP overlay (~24% canvas width).
    9. Collect chunks in `ondataavailable` (start with `recorder.start(1000)`).
    10. Set `_isRecording = true`.

    `stop()` implementation:
    1. Guard: if not recording, return null.
    2. Stop the `requestAnimationFrame` loop (cancel frame ID).
    3. Stop recorder (triggers `onstop`).
    4. In `onstop`: create `Blob(chunks, { type: recorder.mimeType || 'video/webm' })`, create object URL.
    5. Close `AudioContext`.
    6. Stop canvas stream tracks.
    7. Set `_isRecording = false`.
    8. Return `{ previewUrl, mimeType, durationMs }`.

    `cleanup()` implementation: stop recording if active, revoke any lingering object URL.

    Important constraints:
    - Do not mirror local self-view in the recording canvas (no CSS transform needed).
    - If remote stream has no audio tracks when start is called, still start recording (video-only mix).
    - The draw loop must handle `readyState < HAVE_CURRENT_DATA` by drawing a dark placeholder.
  </action>

| Boundary | Description |
|----------|-------------|
| RecordingController → Zustand | Only serializable flags and timestamps enter Zustand; canvas/AudioContext/MediaRecorder/Blob/object URL live inside the controller instance held in a React ref. |
