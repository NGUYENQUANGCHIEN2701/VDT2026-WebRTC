<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend → Redis pub/sub | All published messages are server-originated ServerMessage records serialized by the server; never client-supplied routing targets |
| WS client → PresenceWebSocketHandler | username extracted from JWT principal server-side (session.getAttributes().get("username")); client cannot forge route map key |
| nginx/external → Redis | Redis not exposed on external port in compose production config (dev-only port mapping); pub/sub channels are internal only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02 | Tampering | route:{userId} route map key | mitigate | Key written only by PresenceWebSocketHandler.afterConnectionEstablished using server-extracted username from JWT principal; Redis not externally accessible in prod compose |
| T-06-03 | Tampering | inst:{instanceId} pub/sub channel injection | accept | pub/sub channels are internal; RoutingMessageListener deserializes RoutedEnvelope with tools.jackson ObjectMapper — malformed JSON logs error and returns; server only publishes server-originated ServerMessage records |
| T-06-04 | Information Disclosure | online-users SET + presence:{userId} keys | accept | Keys internal to Redis network; no external query path; snapshot is pushed to authenticated WS sessions only |
| T-06-05 | Elevation of Privilege | Cross-instance session-superseded kick | mitigate | Session kick published with server-extracted username (JWT); target userId comes from Redis route map (server-written), not from client input; RoutingMessageListener delivers only to the matching session in SessionRegistry |
| T-06-06 | Denial of Service | PresenceEventListener broadcastSnapshot spam | accept | Each call to broadcastSnapshot() reads Redis SMEMBERS (O(N)) and writes to local sessions; at demo scale this is acceptable; sweeper's presence-events publish is rate-limited by its 15s schedule |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new package manager installs; all dependencies already in pom.xml; no slopcheck needed |
</threat_model>
