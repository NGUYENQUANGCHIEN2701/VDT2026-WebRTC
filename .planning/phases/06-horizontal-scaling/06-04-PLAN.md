<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Phase 6 complete: Redis pub/sub cross-instance routing (RedisMessageRouter + RedisPresenceService), nginx LB with two backend instances (docker-compose.yml), and a standing automated integration test (CrossInstanceCallTest). The system runs as 2 backends behind nginx; a call connects when caller and callee are on different instances.
  </what-built>

  <how-to-verify>
    1. Start the compose stack (backend only — no frontend in Phase 6):
       docker compose up --build -d postgres redis rabbitmq coturn backend-1 backend-2 nginx

    2. Wait for both backends to be healthy (check with: docker compose ps)

    3. Confirm nginx distributes to both instances:
       curl -s http://localhost:8080/api/health
       Run this 4-5 times. Check docker compose logs backend-1 backend-2 to see that both backends receive requests (round-robin).

    4. Start the frontend separately (if not in compose):
       cd frontend && npm run dev (typically starts on port 5173)

    5. Open two browser windows (or two incognito windows) both pointing to the frontend. Log in as different users (e.g., alice and bob). Both must be registered first via POST /api/auth/register if not already in the DB.

    6. Confirm both users see each other online in the presence list.

    7. From alice's window, call bob. Both should see the ringing state and be able to connect.
       Confirm the call connects (audio/video works).

    8. Check backend logs to confirm alice and bob landed on DIFFERENT instances:
       docker compose logs backend-1 | grep -E "(alice|bob|route)"
       docker compose logs backend-2 | grep -E "(alice|bob|route)"
       (Due to round-robin you may need to refresh a few times to get them on different instances — or connect manually and check route:{userId} in Redis)

    9. Optional Redis verify:
       docker compose exec redis redis-cli GET "route:alice"   — returns "backend-1" or "backend-2"
       docker compose exec redis redis-cli GET "route:bob"     — should return different instance

    10. Tear down: docker compose down
  </how-to-verify>

  <resume-signal>
    Type "demo-ok" if the call connected with both users on different instances.
    Type "demo-fail: [description]" if something did not work so the issue can be investigated.
  </resume-signal>
