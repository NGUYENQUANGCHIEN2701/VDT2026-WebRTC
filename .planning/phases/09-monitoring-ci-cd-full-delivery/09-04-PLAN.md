<objective>
Add a Playwright E2E test that places a real 1-1 video call between two fake-media Chromium browser contexts and asserts the remote `<video>` element renders actual frames, then wire it as a 4th parallel CI job appended to the `.github/workflows/ci.yml` created in Plan 09-03. This directly satisfies INFR-06.

Purpose: Per CONTEXT.md, Playwright E2E design was left fully open — 09-RESEARCH.md's recommendation (and this plan's approach) is to run the E2E test against directly-started backend+frontend processes (GitHub Actions `services:` containers for Postgres/Redis/RabbitMQ, no full `docker compose up`, no coturn/nginx) because two Chromium contexts on the same CI runner negotiate host/server-reflexive ICE candidates without needing TURN relay — this collapses the E2E job's dependency chain and avoids the flakiness of waiting on 8 compose services with healthcheck chains for a test that only needs WS signaling + 2 fake-media peers.
Output: `frontend/e2e/playwright.config.ts`, `frontend/e2e/one-to-one-call.spec.ts`, `@playwright/test` devDependency + `e2e` npm script in `frontend/package.json`, `data-testid` attributes on `CallPage.tsx`'s video elements, and a 4th `e2e` job appended to `.github/workflows/ci.yml`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| e2e CI job -> GitHub Actions service containers | Ephemeral Postgres/Redis/RabbitMQ scoped to the single job run, torn down after; no persistence, no real data |
| Playwright fake-media Chromium -> backend jar (localhost) | Both processes run on the same CI runner; no real network/NAT boundary crossed, consistent with why TURN/coturn is unnecessary here |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-08 | Information Disclosure | `JWT_SECRET` env var in the e2e job | high | mitigate | Set to a clearly-labeled CI-only dummy value (`ci-only-test-secret-min-32-characters-long`), never sourced from `${{ secrets.* }}` or the repo's real `.env` — this job has no legitimate need for the production secret and must never reference it |
| T-09-09 | Information Disclosure | `POSTGRES_PASSWORD`/`RABBITMQ` creds for the e2e job's service containers | low | accept | Dummy values (`vdt`/`vdt`, `guest`/`guest` defaults) scoped to an ephemeral, network-isolated CI job container with no external exposure — acceptable for a throwaway test-run database |
| T-09-10 | Repudiation | Playwright test failures with no diagnostic artifact | low | mitigate | `actions/upload-artifact@v4` with `if: failure()` uploads the Playwright HTML report (including trace files per `trace: 'retain-on-failure'` in the config) so failures are debuggable after the fact |

</threat_model>
