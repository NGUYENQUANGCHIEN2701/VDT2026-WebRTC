<objective>
Patch 3 real anti-spam gaps in the OTP/password-reset flows and split the single root `.env` into environment-specific `.env.local` / `.env.prod` files with new committed templates.

1. `AuthService.requestPasswordReset` has no cooldown (unlike `EmailVerificationService.issueCode`, which already enforces 60s) — every call issues a fresh token and sends an email immediately.
2. No IP-based throttling exists anywhere (no Bucket4j/resilience4j dep, no nginx `limit_req`) for `/api/auth/register`, `/api/auth/resend-verification-otp`, `/api/auth/forgot-password` — an attacker can hammer any of the three from a single IP.
3. `EmailVerificationService.verify()` has no attempt counter — unlimited OTP guesses against a single 6-digit code.
4. `docker-compose.yml`/`docker-compose.prod.yml` both read from one shared root `.env`, which conflates local-dev and production credentials.

Purpose: Close real spam/enumeration/brute-force vectors on public, pre-auth endpoints, and stop local-dev and production secrets from living in the same file.
Output: Cooldown on forgot-password; a reusable Redis-Lua IP rate limiter wired into all 3 public auth endpoints; a max-attempt lockout on email-verification OTP guesses; `.env.local.example` + `.env.prod.example` templates with updated compose/README instructions.
</objective>
